# BMAD: Django Dependency Deferral

**Story:** Epic 7 (Admin Panel)
**Date:** 2025-10-28
**Severity:** Medium (Bloat concern)
**Status:** ⚠️ Defer to Epic 7

---

## Behavior

**Observed**: Django included in `requirements.txt` but unused in Epic 1-6.

**Evidence**:
```bash
$ grep django requirements.txt
Django==4.2.7
# Plus 50+ transitive dependencies
```

**Issue**: Adds ~100MB to Docker images, unused for 6 epics.

---

## Measure

### Recommendation

**Move to Optional Dependencies** (Epic 7 planning):
```bash
# requirements.txt (Epic 1-6)
# Core dependencies only

# requirements-admin.txt (Epic 7)
Django==4.2.7
django-environ==0.11.2
# Admin panel specific deps
```

---

## Act

### Decision: Defer to Epic 7

**Rationale**:
- Not blocking any Epic 1-6 functionality
- Remove during Epic 7 admin panel implementation
- Low priority vs documenting other changes

---

## Document

**Impact**: Medium - 100MB bloat acceptable short-term
**Epic 7 Action**: Move Django to optional deps
